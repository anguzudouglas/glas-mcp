# Skill: document_create_pdf

**Tool:** `document_create_pdf`  
**Version:** 1.0.0

## Purpose
Generate a PDF from HTML+CSS content using WeasyPrint. Supports full CSS3 print layout including page size, margins, headers, footers, tables, and custom fonts (from Google Fonts or bundled).

## Parameters

| Parameter | Type | Default | Notes |
|-----------|------|---------|-------|
| `html_content` | string | **required** | Full HTML document or fragment |
| `filename` | string | `"document.pdf"` | Output filename (`.pdf` added if missing) |
| `page_size` | string | `"A4"` | `A4`, `A3`, `Letter`, `Legal` |
| `margin` | string | `"2cm"` | CSS margin: `"1in"`, `"15mm"` |
| `base_url` | string | `""` | Base URL for resolving relative assets |

## CSS Features Supported by WeasyPrint

✅ CSS3 selectors, flexbox, grid  
✅ `@page` rule (size, margins, headers/footers via `@top-*`, `@bottom-*`)  
✅ `page-break-before`, `page-break-after`, `page-break-inside`  
✅ CSS counters (`counter(page)`, `counter(pages)`)  
✅ Google Fonts (via `@import` — requires internet access)  
✅ SVG images embedded as `<img>` or inline `<svg>`  
✅ Base64-encoded images in `<img src="data:image/...">` — **no external dependencies**  
❌ JavaScript  
❌ CSS animations / transitions  
❌ `position: fixed` (use `@page` for fixed headers/footers)

## Quality Rules

1. **Always use `<!DOCTYPE html>` and `<meta charset="UTF-8">`** — prevents encoding issues.
2. **Embed images as base64** for portability — no broken image links.
3. **Use `@page` with `@bottom-center`** for professional page numbering.
4. **`page-break-after: always`** on cover page / major sections.
5. **Test table widths:** Use `width: 100%` with `border-collapse: collapse`.
6. **For logos/icons:** SVG is preferred — scales perfectly at any DPI.
